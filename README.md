# AI 營養協作平台第二階段互動原型

這是營養科第二階段需求的靜態互動網站原型，整合臨床端、社區端與跨場域照護工作流程。

公開網站：

https://yht5582-source.github.io/ai-icope-bmi-ai-ai-ai/

## 主要功能

- 全期總覽：個案數、高風險、攝食不足、咀嚼吞嚥警訊、風險分布圖。
- 新病人資料輸入：建檔者、病人姓名、評估日期、年齡、性別、場域、ICOPE 功能評估、BMI、白蛋白、體重變化、食慾、攝食量、咀嚼吞嚥狀況。
- ICOPE 功能評估：認知功能、行動能力、營養狀況、視力、聽力、情緒，勾選後自動計算異常項目數並納入 AI 風險分級。
- 全期營養趨勢：依評估日期建立週期標籤，呈現體重、攝食量、白蛋白趨勢。
- AI 餐食辨識分析：上傳餐食照片後顯示示範辨識結果、熱量與三大營養素比例。
- 個案關懷追蹤：依風險分級產生關懷與照會追蹤建議。
- 照護團隊訊息：保留營養師、衛教師、個管師訊息串，可新增團隊訊息。
- 病人資料刪除：個案詳細資料內可刪除病人，儀表板與趨勢會同步更新。

## 資料保存

目前是前端原型，病人資料儲存在使用者瀏覽器的 `localStorage`，不會寫入後端資料庫。

因此：

- 同一台電腦、同一瀏覽器重新整理後，資料會保留。
- 換瀏覽器、清除網站資料或使用無痕視窗時，資料可能不會保留。
- 未來若要正式上線，建議串接 HIS / ICOPE / 後端資料庫。

```bash

```

開啟：

```text
http://localhost:4173
```

## 檔案結構

```text
index.html                         # GitHub Pages 單檔部署版
styles.css                         # 本機開發版樣式
script.js                          # 本機開發版互動邏輯、風險分級、趨勢圖、localStorage
assets/nutrition-care-visual.svg   # 本機開發版首頁視覺圖
```

## 部署

此專案為純靜態網站，可部署於 GitHub Pages。

目前 GitHub Pages 使用：

- Repository: `yht5582-source/ai-icope-bmi-ai-ai-ai`
- Branch: `main`
- Folder: `/ (root)`

## 注意事項

餐食照片分析目前為互動原型，會套用示範餐別資料產生熱量與三大營養素結果；正式版本需串接影像辨識模型與食物營養資料庫。
